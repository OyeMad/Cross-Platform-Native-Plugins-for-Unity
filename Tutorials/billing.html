Billing Services (In-App Purchases)

This feature offers allows you to monetize your app with In-App Purchases 

Get Billing Products

Get details of In-App purchase products set in corresponding stores

Receipt Validation

Kick back hackers by validating your purchases with external or app store servers

Buy Product

Buy a virtual item by providing product information

Restore Purchases

Restore non-consumable purchases to make them available on any device 

Setup

Finish the setup on the native platforms before using the features.

Once you create the In-App products on respective stores, take a note of the product identifiers to fill in NPSettings Editor.

iOS Android
Unity Editor - NPSettings

Initial Steps

1. Request for Billing Products

This should be the first call to use rest of the features. This fires an event once finished.

2. Restore Purchases

Once after requesting the billing products, if you have any non-consumable products, you can call this method to restore the purchases of the  user if he/she has purchased earlier.

NPBinding.Billing.RequestForBillingProducts

This needs to be the first call before using any of the other methods of this feature. Pass list of products you set in NPSettings which can be fetched by using NPSettings.Billing.Products.
NPBinding.Billing.RequestForBillingProducts method call fetches the details of products requested from store. These details will be used internally as well as you can use it for displaying more details about an In-App purchase. 

Pro Tip : You can even pass the Billing Products which you can construct at runtime. This will be useful when you need to fetch the billing product details from your server API. Create a list a BillingProduct instances, fill in the details and pass the list to RequestForBillingProducts.

Initiate restoring the purchases by below code

                          
NPBinding.Billing.RequestForBillingProducts(NPSettings.Billing.Products);
// At this point you can display an activity indicator to inform user that task is in progress

Register the callback to get the details once the request fetch from store is finished.


Register the event in OnEnable
Billing.DidFinishRequestForBillingProductsEvent	+= OnDidFinishProductsRequest; 


Un-Register the event in OnDisable
Billing.DidFinishRequestForBillingProductsEvent	-= OnDidFinishProductsRequest;


Implement Callback

private void OnDidFinishProductsRequest (BillingProduct[] _regProductsList, string _error)
{
    // Hide activity indicator

    // Handle response
    if (_error != null)
    {        
        // Something went wrong
    }
    else 
    {    
        // Inject code to display received products
    }
}

Register for  Billing.DidFinishRequestForBillingProductsEvent event and call NPBinding.Billing.RequestForBillingProducts.

public void RequestBillingProducts ()
{
    NPBinding.Billing.RequestForBillingProducts(NPSettings.Billing.Products);

Restore Purchases (Required only if you have Non-Consumable Products)

  • Description
  • Usage
  • Callbacks
  • Example

Your Game/App needs to provide an option to users for restoring any previously purchased non-Consumable products. This can be the case when user re-installs the app or signing back from a different store account. 

Only non-consumable products are restorable. Consumable products won't be able to restore.

Note: If your app has Non-Consumable products, you must implement a mechanism to restore old purchases. In fact, your app might get rejected if you fail to do so.

Initiate restoring the purchases by below code :

                          
private void RestoreCompletedTransactions ()
{
      NPBinding.Billing.RestoreCompletedTransactions ()
}

Register for callbacks to get the list of products restored.

Billing.DidFinishRestoringPurchasesEvent

Registering to this callback gives list of purchases that are purchased earlier by the user.


Register event in OnEnable

                          
Billing.DidFinishRestoringPurchasesEvent        += OnDidFinishRestoringPurchases;

Register event in OnEnable

                          
Billing.DidFinishRestoringPurchasesEvent        -= OnDidFinishRestoringPurchases;

Implement callback

                          
private void OnDidFinishRestoringPurchases (BillingTransaction[] _transactions, string _error)
{
    Debug.Log(string.Format("Received restore purchases response. Error = {0}.", _error));

    if (_transactions != null)
    {                
        Debug.Log(string.Format("Count of transaction information received = {0}.", _transactions.Length));

        foreach (BillingTransaction _currentTransaction in _transactions)
        {
            Debug.Log("Product Identifier = "         + _currentTransaction.ProductIdentifier);
            Debug.Log("Transaction State = "        + _currentTransaction.TransactionState);
            Debug.Log("Verification State = "        + _currentTransaction.VerificationState);
            Debug.Log("Transaction Date[UTC] = "    + _currentTransaction.TransactionDateUTC);
            Debug.Log("Transaction Date[Local] = "    + _currentTransaction.TransactionDateLocal);
            Debug.Log("Transaction Identifier = "    + _currentTransaction.TransactionIdentifier);
            Debug.Log("Transaction Receipt = "        + _currentTransaction.TransactionReceipt);
            Debug.Log("Error = "                    + _currentTransaction.Error);
        }
    }
}

Buy Product

  • Description
  • Usage
  • Callbacks
  • Example

Once you fetch all your billing products and display them you can let users to buy products and make some money!

Check first if the product is already purchased. This call will only work for non-consumable products. For consumable products it always returns false.

                          
if (NPBinding.Billing.IsProductPurchased(_product.ProductIdentifier))
{
	// Show alert message that item is already purchased
	Debug.Log("Product already purchased"); 
	return;
}

Call BuyProduct by passing a Billing Product

                          
public void BuyItem (BillingProduct _product)
{
    if (NPBinding.Billing.IsProductPurchased(_product.ProductIdentifier))
    {
        // Show alert message that item is already purchased

        return;
    }

    // Call method to make purchase
    NPBinding.Billing.BuyProduct(_product);

    // At this point you can display an activity indicator to inform user that task is in progress
}

Register for callback events to get the purchase status.

Registering to this callback gives the transaction status. It can be a failed purchase or failed verification(happens when there are fraud transactions).


Register event in OnEnable

                          
Billing.DidFinishProductPurchaseEvent            += OnDidFinishTransaction;

Register event in OnEnable

                          
Billing.DidFinishProductPurchaseEvent            -= OnDidFinishTransaction;

Implement callback

                          
private void OnDidFinishTransaction (BillingTransaction _transaction)
{
    if (_transaction != null)
    {
		// Check if the transaction is verified successfully. By default we verify with Apple
        if (_transaction.VerificationState == eBillingTransactionVerificationState.SUCCESS)
        {
			// Check if the transactions is successful
            if (_transaction.TransactionState == eBillingTransactionState.PURCHASED)
            {
                // Your code to handle purchased products
            }
        }
    }
}

FAQ

Why IsProductPurchased always returning FALSE?

Only non-consumable products are restorable. Make sure you call RestorePurchases at start before using IsProductPurchased

What needs to be taken care before submission to Apple iOS Store?

1. If you are using Non-Consumable Products, make sure you have a button for Restoring Purchases in your app. If this is not added, many chances for your app getting rejected.

2. Once after exporting from Unity, never change the build version in Xcode. This makes the Verification to fail as it will have mismatch in bundle versions. So, make sure you NEVER change the build version after exporting from Unity.

My Billing Products are fetched from a server. How to handle this situation?

You can create a list of BillingProducts at runtime as well. Once you create a list, after filling the details, use it similar to how you use NPSettings.Billing.Products.

Why RequestForBillingProducts returning empty list? 

1. Make sure you filled in the Billing Products in NPSettings.
2. Finish your Tax Documentation on iTunes Connect or iOS Developer Portal. If it's not finished, it will not return any product details.

On iOS, always store login is shown. What could be the reason?

Incase, if you disable the default verification, you need to manually finish the transaction. Please check this section on how to manually finish the transaction.

On Android, for Consumable products do I need to call any Consume API?

Not required. We internally consume automatically for consumable products. Awesome - Right? ;)

On iOS,  Sandbox user details are not getting authenticated. Why?

Make sure you sign out of real App Store account from settings and only enter the login details when the app prompts with a login window. Don't enter sandbox user details in Settings. This makes the sandbox user account to get invalidated.

NPSettings for Billing Feature

In your Unity project, If Billing is not enabled, enable by below steps.
1. Open NPSettings and select Application Settings. 
2. Under supported features section, Enable Uses Billing checkbox and Save (see button at bottom).

Once you have Billing feature enabled, select Billing Settings. Let us take a brief look into it:

1. Products: Add the product identifiers for each platform you created in Registering your Products section. Later on, you can call NPSettings.Billing.Products to access this data from your code.

2. iOS Settings: Here, first thing you will notice is “Supports Receipts Validation”. This optional feature allows you to verify receipts of completed transactions. 

Enabling receipt validation, adds one more level of security to avoid unauthorised purchases. By default, our plugin contacts Apple Server to verify transaction receipts. However you are free to override this behaviour, by providing URL of the server to which receipt data has to be sent for verification. 
For more information, see Validating Receipts With App Store.
 
3. Android Settings: You need to provide public key provided by Google Play services. This key is used for verifying signatures. For information on how to get Public Key, read Securing Your Application. 
Public key value is REQUIRED to use Billing feature on Android. 

Thats it! Save all your NPSettings changes by clicking Save button at bottom of the inspector.
